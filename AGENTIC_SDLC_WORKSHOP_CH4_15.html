Agentic SDLC Team Coaching Workshop

Workshop design for coaching a team through Chapters 4-15: from PRD and architecture to code, tests, CI/CD, production readiness, observability, incidents, retrospectives, and agent team operations.

Ch 4-15 4 Days PM + Dev + Tester + DevOps Human + AI Agent Git Artifacts

Workshop Outcomes

Team Operating Model

The team can explain which work belongs to Human PM, Developer, Tester, DevOps, SRE, and AI Agents.

Git-Based Traceability

Every important artifact is stored in Git: PRD, ADR, tests, CI/CD, runbooks, incident reports, and CLAUDE.md updates.

Feature Delivery

Each group delivers one feature from requirement to production readiness using agentic workflows.

Target outcome: one feature moves from PRD → architecture → code → tests → CI/CD → launch readiness → observability → incident learning.

Facilitator Guide

Session Rhythm

Explain concept with figure.
Show example prompt.
Assign human role and AI agent.
Run hands-on exercise.
Review artifact.
Commit artifact to Git.
Reflect: what becomes a rule, skill, or runbook?

Workshop Rules

Every exercise ends with a committed artifact
No AI direct push to main
Every AI artifact gets human review
Every architecture choice gets ADR or note
Every production change gets rollback path
Every incident learning updates memory or process

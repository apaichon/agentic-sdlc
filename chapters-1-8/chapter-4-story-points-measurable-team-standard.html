Story Points: Measurable Team Standard

Estimate by observable behavior, uncertainty, integration risk, and team-agreed standards.

Chapter 4 · After 4.4

From Vague Story to Measurable Story

Weak input

"Build consolidated chat."

  • No acceptance criteria
  • No target latency
  • No known data stores
  • No integration boundary
  • No test data

Measurable input

"As a support agent, I can see LINE, Facebook, and WhatsApp messages in one Next.js inbox within P95 2s, persisted to MongoDB, routed by RabbitMQ, cached in Redis, and measured in ClickHouse."

Story Point Checklist

✓ User value is clear
✓ Behavior is observable
✓ Given/When/Then exists
✓ Dependency is known
✓ Test data is known
✓ Risk is visible
✓ Team standard is applied
✓ Done means verified

Tips การให้คะแนน: นับ Architecture Layers + Deployment Stages

มุมที่ต้องดู / LensChecklist ที่ต้องถามผลต่อ Scoreตัวอย่าง Consolidated Chat
Architecture layers แตะกี่ layer: UI, API, service, queue, cache, database, analytics, monitoring. ยิ่งหลาย layer ยิ่งมี integration risk และ score สูงขึ้น. Next.js + Go API + RabbitMQ + MongoDB มักเริ่มที่ 5 points.
DB schema change มี field, index, collection shape, compatibility หรือ rollback ไหม. เพิ่ม risk เพราะกระทบ persistent data. เพิ่ม `delivery_status` และ index `conversation_id` ใน MongoDB.
Migration ต้อง backfill, idempotency, old data, deploy order หรือ rollback plan ไหม. มักดันจาก 3 → 5 หรือ 5 → 8. Backfill WhatsApp messages เก่าเข้า normalized message schema.
Cache behavior มี TTL, invalidation, fallback, race condition หรือ stale data ไหม. เพิ่ม uncertainty ถ้ายังไม่มี pattern เดิม. Redis active assignment cache พร้อม MongoDB fallback.
Backup / restore restore เคยทดสอบไหม, RPO/RTO ชัดไหม, data recover ได้ไหม. เพิ่ม operational scope และ release checklist. ทดสอบ MongoDB backup restore ก่อนเปิด migration.
Monitoring มี metrics, logs, traces, dashboard, alerts และ runbook ไหม. เพิ่มงาน แต่ลด release risk. Alert RabbitMQ queue depth, Go error rate, ClickHouse ingest lag.
Security มี auth, permission, secrets, PII, webhook signature หรือ audit log ไหม. อาจเพิ่ม score หรือบังคับ split story. Verify WhatsApp webhook signature และ mask customer phone numbers.

Team rule: Story points measure relative effort and uncertainty, not hours.
Planning rule: If the team cannot write measurable done criteria, do not estimate yet.

Step-by-step Workflow

1

Create epic & ownership map

แบ่ง workstream ของ PM, Dev, Tester, DevOps พร้อม owner และ artifacts.

2

Create role branches

ใช้ branch type ตาม role เช่น docs/, feature/, test/, ops/.

3

Create worktrees

แยก working directory สำหรับแต่ละ role เพื่อลด file conflict.

4

Merge to dev

ทุก role branch เปิด PR เข้า dev เพื่อรวมงานและรัน integration checks.

5

Open draft PR early

เปิด PR ตั้งแต่ต้นเพื่อเห็น scope, reviewers, risk และ checklist.

6

Promote environments

dev → staging → main → prod พร้อม CI, UAT, approval และ rollback plan.

Environment Branch Flow

devIntegration branch สำหรับ role branches ทั้งหมด ใช้รัน unit, integration, contract tests และ resolve conflicts ก่อนส่งต่อ.
stagingRelease candidate / UAT branch ใช้ตรวจ end-to-end flow, data migration, monitoring และ rollback rehearsal.
mainProduction-ready source of truth ต้อง protected, require PR, CI green, approvals และ signed release note.
prodProduction deployment branch หรือ release tag ใช้ deploy จริง, hotfix และ incident traceability.
PM: docs/* → dev
Dev: feature/* → dev
Tester: test/* → dev
DevOps: ops/* → dev

Test Plan Prompt

Task:
Draft a test plan for the WhatsApp Adapter feature.

Action:
Read the PRD, user stories, architecture docs, implementation task list, and code conventions. Create a risk-based test strategy, test scenarios, test data plan, automation priority, and release confidence checklist.

Goal:
Produce test artifacts that Tester Agent, Developer Agent, and Human QA can use to verify behavior before release.

Context:
- Feature: WhatsApp Adapter for Unified Inbox
- UI: Next.js inbox
- Backend: Go API Gateway and Message Router
- Queue: RabbitMQ
- Storage: MongoDB
- Cache: Redis
- Analytics: ClickHouse

Required Output:
1. Test strategy summary
2. Risk matrix
3. Test scenarios by level
4. Given/When/Then scenario table
5. Test data plan
6. Automation priority
7. PostToolUse verification plan
8. Release confidence checklist

Skill: /draft-test-plan

PurposeDraft risk-based test strategy, test scenarios, test data plan, automation priority และ release confidence checklist.
Triggerใช้เมื่อ PRD, acceptance criteria, architecture และ implementation plan พร้อมแล้ว.
InputsPRD, user stories, acceptance criteria, architecture docs, implementation tasks, known risks.
WorkflowRead docs, identify risk, map risk to test level, draft Given/When/Then, define test data, prioritize automation, create release checklist.
Quality RulesHigh-risk behavior must have automated coverage, contract boundaries tested, performance measurable, security and recovery scenarios included.

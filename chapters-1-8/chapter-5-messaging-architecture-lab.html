Architecture Prompt

Task:
Design messaging architecture for a consolidated chat / Unified Inbox platform.

Action:
Use the tech stack and produce architecture decisions, message flow, schemas, risks, and validation checklist.

Goal:
Create reviewable architecture docs and ADR for Developer, Tester, and DevOps Agents.

Context:
- UI: Next.js Unified Inbox
- Backend: Go API Gateway and Message Router
- Routing: RabbitMQ fanout exchange
- Storage: MongoDB UnifiedMessage collection
- Cache: Redis session cache and rate limiter
- Analytics: ClickHouse event pipeline

Required Output:
1. End-to-end message flow
2. RabbitMQ exchange and queue design
3. MongoDB UnifiedMessage schema
4. Redis key design
5. ClickHouse analytics event schema
6. Failure isolation, retry, idempotency, backpressure
7. Monitoring signals
8. ADR with alternatives and consequences

Skill: /design-messaging-architecture

PurposeDesign messaging architecture สำหรับ multi-channel systems ที่มี queue, storage, cache, analytics และ operational safeguards.
Triggerใช้เมื่อ feature แตะหลาย channels, queue topology, storage, cache, analytics หรือ ADR.
InputsBusiness goal, channels, tech stack, data retention, latency/throughput target.
WorkflowMap message flow, design RabbitMQ, define MongoDB schema, Redis keys, ClickHouse schema, failure isolation, idempotency, monitoring, ADR.
Quality RulesEach component has one responsibility, writes are idempotent, analytics failure must not block message storage, monitoring includes queue depth and ingest lag.
